# App Store Screenshot Requirements Guide

Complete guide for creating App Store screenshots for Plena meditation tracking app.

---

## iOS App Screenshots

### Required Screenshots

**iPhone 6.7" Display** (iPhone 14 Pro Max, 15 Pro Max, etc.)

- **Size:** 1290 x 2796 pixels
- **Required:** Yes
- **Format:** PNG or JPEG
- **Note:** This is the largest iPhone size, covers all smaller sizes

**iPhone 6.5" Display** (iPhone 11 Pro Max, XS Max, etc.)

- **Size:** 1242 x 2688 pixels
- **Required:** Yes
- **Format:** PNG or JPEG

**iPhone 5.5" Display** (iPhone 8 Plus, etc.)

- **Size:** 1242 x 2208 pixels
- **Required:** Optional but recommended
- **Format:** PNG or JPEG

### iPad Screenshots (If Supporting iPad)

**iPad Pro 12.9" (3rd generation)**

- **Size:** 2048 x 2732 pixels
- **Required:** Only if app supports iPad
- **Format:** PNG or JPEG

**Note:** Plena is primarily an iPhone app. iPad screenshots may not be necessary unless iPad support is added.

## Screenshot Content Guidelines

### Recommended Screenshot Order

**iOS App Screenshots:**

1. **Screenshot 1: Main Meditation Screen**

   - Show real-time heart rate, HRV, respiratory rate
   - Display stress zone indicators (Calm/Optimal/Elevated)
   - Clean, focused interface
   - Text overlay (optional): "Track meditation with real-time biometrics"

2. **Screenshot 2: Stress Zone Classification**

   - Highlight zone indicators in action
   - Show different zones (blue/green/orange)
   - Demonstrate real-time feedback
   - Text overlay (optional): "Understand your body's response instantly"

3. **Screenshot 3: Dashboard with Statistics**

   - Show session count, streak, total time
   - Display session frequency chart
   - Highlight progress tracking
   - Text overlay (optional): "Track your progress over time"

4. **Screenshot 4: Data Visualization**

   - Show interactive graphs
   - Display multiple sensors (HR, HRV, breathing)
   - Demonstrate time range views
   - Text overlay (optional): "See your meditation journey visualized"

5. **Screenshot 5: Session Summary**

   - Show post-session statistics
   - Display zone distribution
   - Highlight insights
   - Text overlay (optional): "Detailed insights after every session"

6. **Screenshot 6: Apple Watch Integration** (if space)
   - Show Watch app interface
   - Demonstrate wrist-based tracking
   - Highlight convenience
   - Text overlay (optional): "Start sessions from your Apple Watch"

**watchOS App Screenshots:**

1. **Screenshot 1: Active Session**

   - Show real-time sensor readings on Watch
   - Display heart rate with zone indicator
   - Clean, readable Watch interface
   - Text overlay (optional): "Track meditation on your wrist"

2. **Screenshot 2: Multiple Sensors**
   - Show scrolling through sensors (HR, HRV, breathing)
   - Demonstrate zone indicators
   - Highlight Watch-specific features

## Screenshot Design Best Practices

### Do's

✅ **Show Real App Interface**

- Use actual app screenshots, not mockups
- Ensure screenshots reflect current app version
- Use real data (or anonymized realistic data)

✅ **Highlight Key Features**

- Make important features clearly visible
- Use arrows or annotations if needed (sparingly)
- Show unique features (stress zones, HRV tracking)

✅ **Keep Text Minimal**

- If using text overlays, keep them short
- Use clear, readable fonts
- Don't obscure important UI elements

✅ **Maintain Consistency**

- Use consistent style across all screenshots
- Similar color schemes and layouts
- Professional appearance

✅ **Test on Actual Devices**

- Take screenshots on real devices when possible
- Verify readability and clarity
- Check that text is legible at App Store size

### Don'ts

❌ **Don't Overload with Text**

- Avoid long paragraphs
- Don't cover important UI elements
- Keep descriptions concise

❌ **Don't Use Mockups**

- Avoid generic design mockups
- Use actual app screenshots
- Show real functionality

❌ **Don't Show Placeholder Data**

- Use realistic data examples
- Avoid obvious placeholders (e.g., "XXX BPM")
- Make data look authentic

❌ **Don't Clutter the Screenshots**

- Keep interfaces clean
- Highlight one feature per screenshot
- Maintain visual hierarchy

## Text Overlay Guidelines

### When to Use Text Overlays

- **First Screenshot:** Tagline or main value proposition
- **Feature Screenshots:** Brief feature descriptions
- **Highlight Unique Features:** Stress zones, HRV insights, etc.

### Text Overlay Best Practices

**Style:**

- Large, bold, readable fonts
- High contrast with background
- Positioned to not obscure key UI
- Consistent styling across screenshots

**Content Examples:**

- "Track meditation with real-time biometrics"
- "Understand your body's response"
- "See your progress visualized"
- "Start sessions from your Apple Watch"
- "Data-driven meditation insights"

**Avoid:**

- Long sentences
- Technical jargon
- Multiple fonts/styles
- Text that covers critical UI

## Screenshot Tools & Resources

### Built-in Tools

**iPhone/iPad:**

- Use device screenshot feature (Side button + Volume Up)
- Screenshots are automatically saved to Photos
- Edit in Photos app or third-party tools

**Simulator (for Testing):**

- File → Save Screen (⌘+S)
- Useful for testing before final device screenshots
- Note: Use real device screenshots for submission

## Quality Checklist

Before submitting screenshots, verify:

- [ ] All screenshots are correct sizes (pixel-perfect)
- [ ] Screenshots show actual app functionality
- [ ] Text overlays are readable and professional
- [ ] No placeholder data visible
- [ ] Key features are clearly highlighted
- [ ] Consistent style across all screenshots
- [ ] Screenshots work well at App Store display size
- [ ] No private or sensitive data visible
- [ ] Screenshots reflect current app version
- [ ] Both iOS and watchOS screenshots included (if applicable)

## Testing Your Screenshots

### Before Submission

1. **View at Actual Size**

   - Check how screenshots look at App Store display size
   - Ensure text is readable
   - Verify features are clearly visible

2. **Test on Different Devices**

   - View on iPhone (if possible)
   - Check how screenshots appear in App Store app
   - Verify consistency

3. **Get Feedback**

   - Show screenshots to others
   - Ask if key features are clear
   - Verify messaging is understandable

4. **Compare to Competitors**
   - Review similar app screenshots
   - Identify what works well
   - Differentiate your approach
